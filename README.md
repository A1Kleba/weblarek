# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

###### Модели данных

###### Класс Cart
Класс для управления корзиной покупок. Позволяет добавлять, удалять и анализировать товары в корзине.

свойства:
`items: IProduct[]` — массив товаров, добавленных в корзину (хранится внутри класса, приватный).

Конструсктор создаёт новую корзину с пустым массивом товаров.

Методы:
`setItems(items: IProduct[]): void` - заменяет текущее содержимое корзины на переданный массив товаров.
Параметры:
`items` — массив объектов IProduct для загрузки в корзину.

`getItems(): IProduct[]` -возвращает текущий массив товаров в корзине.
Возвращает:
Массив объектов IProduct.

`addItem(product: IProduct): void` - добавляет товар в корзину.
Параметры:
`product` — объект IProduct, который нужно добавить.

`removeItem(product: IProduct): void` - удаляет товар из корзины по его id.
Параметры:
`product `— объект IProduct, который нужно удалить.
Примечание: сравнивает товары по полю id.

`clear(): void` -очищает корзину — удаляет все товары.

`getTotalPrice(): number` - считает общую стоимость всех товаров в корзине.
Возвращает:
Число (сумма цен всех товаров).
Примечание: если у товара price равен null/undefined, он учитывается как 0.

`getCount(): number`- возвращает количество товаров в корзине.
Возвращает:
Число (количество товаров).

`hasItem(productId: string):` boolean - проверяет, есть ли товар с указанным id в корзине.
Параметры:
`productId`— строка с идентификатором товара.
Возвращает:
true, если товар найден;
false, если товара нет в корзине.

###### Класс Customer
Модель данных покупателя для хранения и валидации контактной информации и способа оплаты.

Свойства:
`payment: IBuyer["payment"]` — способ оплаты (по умолчанию: "card").
`email: string` — email‑адрес покупателя.
`phone: string` — номер телефона.
`address: string` — адрес доставки.

Все свойства приватные, доступ через методы класса.

Конструктор инициализирует модель со значениями по умолчанию.

Методы:
`setData(data: Partial<IBuyer>): void` - частично обновляет данные покупателя. Для каждого поля применяется оператор ??, чтобы сохранить текущее значение при отсутствии нового.
Параметры:
`data` — объект с полями для обновления (необязательные поля).

`getData(): IBuyer`- возвращает полный объект с текущими данными покупателя.
Возвращает:
Объект типа IBuyer со всеми полями.

`clear(): void` - сбрасывает все поля к значениям по умолчанию (как в конструкторе).

`validate(): { payment?: string; email?: string; phone?: string; address?: string }` - проверяет валидность всех полей и формирует объект с сообщениями об ошибках.
Возвращает:
Объект, где ключи — поля с ошибками, значения — текстовые описания ошибок.

Правила валидации:
payment: не пустое и не состоит только из пробелов.
email: обязательное поле; формат проверяется регулярным выражением /^\S+@\S+\.\S+$/.
phone: обязательное поле; длина ≥ 10 символов.
address: обязательное поле; длина > 5 символов.

`isValid(): boolean` - проверяет, что все поля валидны (нет ошибок валидации).
Возвращает:
true, если валидация прошла успешно (объект ошибок пуст);
false, если есть хотя бы одна ошибка.

###### Класс ProductCatalogModel
Модель каталога товаров для управления ассортиментом и текущим выбранным товаром.

Свойства:
`products: IProduct[]` — массив всех товаров в каталоге (приватное).
`selectedProduct: IProduct | null` — текущий выбранный товар (приватное, может быть null).

Конструктор:
`constructor(products: IProduct[] = [])`
Параметры:
products (опционально) — начальный массив товаров. По умолчанию: пустой массив [].

Методы:
`setProducts(products: IProduct[]): void` - заменяет текущий массив товаров на новый.
Параметры:
products — массив объектов IProduct для загрузки в каталог.

`getProducts(): IProduct[]` - возвращает полный список товаров каталога.
Возвращает:
Массив объектов IProduct.

`getProductById(id: string): IProduct | undefined` - находит товар по уникальному идентификатору.
Параметры:
id — строка с идентификатором товара.
Возвращает:
Объект IProduct, если товар найден;
undefined, если товар не существует.

`setSelectedProduct(product: IProduct): void` - устанавливает текущий выбранный товар.
Параметры:
product — объект IProduct, который нужно сделать выбранным.

`getSelectedProduct(): IProduct | null` - возвращает текущий выбранный товар.
Возвращает:
Объект IProduct, если товар выбран;
null, если ни один товар не выбран.

##### Слой коммуникации
#####  Класс CommunicationService
Назначение:
отвечает за взаимодействие клиентского приложения с серверной частью:

Свойства:
`api: Api` — экземпляр HTTP‑клиента (приватное, задаётся в конструкторе).
Конструктор
`constructor(api: Api)`
Параметры:
api — готовый экземпляр класса Api, обеспечивающий HTTP‑взаимодействием (GET, POST и др.).

Методы:
`getProductList(): Promise<IProduct[]>` - получает список товаров из API.
Возвращает:
`Promise<IProduct[]>` — массив товаров типа IProduct[].
При ошибке возвращает пустой массив [].

Логика:
-Выполняет GET‑запрос к /product/.
-Извлекает поле items из ответа (IApiProductsResponse).
-Если items отсутствует, возвращает [].
-При исключении логгирует ошибку и возвращает [].

`sendOrder(order: IOrder): Promise<IApiOrderResponse>` - отправляет данные заказа на сервер.
Параметры:
order — объект типа IOrder с данными заказа.
Возвращает:
`Promise<IApiOrderResponse>` — ответ сервера типа IApiOrderResponse.
При ошибке возвращает объект { total: 0 }.

Логика:
-Выполняет POST‑запрос к /order/ с телом order.
-При исключении логгирует ошибку и возвращает дефолтный ответ.
